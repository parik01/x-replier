# Ручной тест X-replier

Автоматические тесты не могут проверить реальный DOM и публикацию X. Тестируйте на отдельном аккаунте.

## Подготовка

1. Откройте `chrome://extensions`, включите Developer mode и загрузите папку репозитория через Load unpacked.
2. Откройте `https://x.com/home`, войдите в тестовый аккаунт.
3. В popup добавьте ограниченный OpenAI API key и нажмите «Проверить OpenAI».

## Draft

1. Выберите Draft, лимит 2 и нажмите «Запустить».
2. Убедитесь, что выбран подходящий пост, а текст полностью вставлен.
3. Нажмите «Следующий»: публикация не должна увеличивать счётчик отправок.
4. Запустите снова, отправьте черновик вручную в X и нажмите «Отправлено» на панели.
5. Убедитесь, что счётчик увеличился только после этого действия.
6. Проверьте «Пропустить» и «Стоп».

## Auto

1. Используйте отдельный тестовый аккаунт, `dailyLimit=1`, задержку 30–45 секунд.
2. Включите Auto и отдельно подтвердите понимание риска.
3. Запустите, затем во время таймера нажмите «Стоп»: публикация не должна уйти.
4. Повторите тест и дождитесь публикации.
5. Убедитесь, что счётчик увеличился только после закрытия редактора без ошибки.
6. Повторно запустите реплаер: тот же post ID не должен попасть в очередь.

## При ошибке

Сделайте скриншот панели X-replier, сообщения X и консоли расширения. Не используйте Auto до исправления причины.
