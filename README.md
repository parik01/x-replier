# X-replier

Личное Chrome-расширение для X/Twitter: оно находит подходящие видимые посты, создаёт короткие ответы через OpenAI и подставляет их в нативный редактор X.

## Режимы

- **Draft** — режим по умолчанию. Расширение вставляет черновик, а вы вручную отправляете его в X и подтверждаете это на панели.
- **Auto** — после проверенной вставки и случайной задержки расширение нажимает «Ответить». Режим выключен по умолчанию, требует отдельного подтверждения и предназначен только для тестового/ограниченного личного использования.

## Что проверено автоматически

- валидация настроек, лимитов и задержек;
- состояния `drafted`, `sent`, `skipped`, `failed`, `unknown`;
- Draft не увеличивает дневной счётчик;
- подтверждённый `sent` увеличивает его;
- очистка и ограничение ответа модели;
- истечение глобальной блокировки;
- обрезка истории.

Запуск: `npm test` и `npm run check`.

## Что требует ручного теста

Реальный DOM X, вставка в React-редактор, закрытие формы после публикации и реакция X на автоотправку. Полный чек-лист — в [docs/MANUAL_TEST.md](docs/MANUAL_TEST.md).

## Установка

1. Скачайте или клонируйте репозиторий.
2. Откройте `chrome://extensions`, включите Developer mode.
3. Нажмите **Load unpacked** и выберите папку проекта.
4. Откройте `https://x.com/home`, нажмите иконку расширения, добавьте OpenAI API key и используйте «Проверить OpenAI».
5. Начните с режима Draft и тестового аккаунта.

## Ограничения и конфиденциальность

- Проект работает только с отрисованными постами текущей ленты; X может менять DOM и селекторы.
- Расширение не ставит лайки, не подписывается, не пишет личные сообщения и не обходит лимиты X.
- Текст выбранного поста отправляется в OpenAI только для генерации ответа.
- API key хранится в `chrome.storage.local`, не передаётся в content script и не логируется. Такое хранение подходит для личного расширения, но не равно защищённому хранилищу секретов.
- Не включайте Auto, пока не завершите ручной тест и не убедитесь, что правила X и ваш сценарий это допускают.
